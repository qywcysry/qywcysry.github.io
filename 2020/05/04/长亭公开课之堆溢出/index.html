什么是堆

栈通常用于为函数分配固定大小的局部内存，而堆是可以根据运行时的需要进行动态分配与释放的内存，大小可变。Malloc/New-Free/Delete，两组接口，分别是申请内存的两种方式，C与C++。

堆的实现重点关注内存块的组织和管理方式，尤其是空闲内存块。重点在于如何提高分配和释放效率/如何降低碎片化，提高空间利用率。

常见堆实现

dlmalloc-通用分配器

ptmalloc2-glibc，基于dlmalloc fork出来，支持多线程。

jemalloc-FreeBSD，Firefox，Android

tcmalloc-Google Chrome

libumen-Solaris

segment heap-Win10

ptmalloc2的多线程支持：

不同的线程维护不同的堆，称为per thread arena

主线程创建的堆，称为main arena

arena数量受到CPU核数限制：32位系统的arena数量上限=2x核数，64位系统arena数量上限=8x核数。

glibc的堆管理实现

arena

该结构体指的是堆内存区域本身而并非结构，主线程的main arena通过sbrk创建，其他线程arena通过mmap创建。

malloc_state

该结构体管理arena的核心结构，包含堆的状态信息，bins链表等。main arena对应的mallloc_state结构存储在glibc的全局变量中。其他线程arena对应的malloc_state存储在arena本身中。

bins

bins用来管理空闲内存块，通常使用链表结构来进行组织。

chunks

Bins结构

用来管理和组织空闲内存块的链表结构，根据大小状态分为许多。

Fast bins，小的chunk。

small bins，中等大小的chunk

large bins，较大的chunk

unsorted bins，存放未整理chunk。

建议看glibc源码

fastbin attack

Fast bins：还是自己写程序对照源码看。

  1. 大小，32位是16-64字节，64位是32-128字节。
  2. 相同大小的chunk放在一个bin中。
  3. 单向链表，fd指向上一个free chunk。
  4. 后进先出 First in last out。先释放的chunk反而先被申请（同等大小，后释放，先利用）。
  5. 相邻的空闲fast bin chunk不会被吞并（P标志不改变）。
  6. 当chunk被free时，不会清理PREV_INUSE标志（还保留原来的数据）。

内存中两位一个字节。0X41414141，一个字节一个ASCII码。NMP怎么表示。

malloc和free的工作流程

malloc()工作流程
  1. 如果size<max_size，在fast bins中寻找发射台 chunk，找到则结束。
  2. 如果size in_smallbin_range，在small bins中寻找small chunk。
  3. 如果size not in_smallbin_range,合并所有fastbin的chunk
  4. 循环a：检查unsorted bin中的last_ramainder,满足一定条件分裂之，剩余的chunk标记为新的last_remainder。
  5. 循环b：在unsorted bin中搜索，同时进行整理。如果遇到精确大小则返回，否则把当前chunk整理到small/large bins中去。
  6. 循环c：在small bin和large bin中搜索最合适的chunk（不一定是精确大小）
  7. 使用top chunk。
free()的工作流程
  1. 如果size < max fast,放入fast bin，结束
  2. 如果前一个chunk是free的，unlink前面的chunk，并合并两个chunk放入unsorted bin。
  3. 如果后一个chunk是top chunk，则将当前chunk并入top chunk。
  4. 如果后一个chunk是free的，unlink后面的chunk，合并两个chunk并放入unsorted bin。
  5. 前后都不是free的，放入unsorted bin.
Fast bin利用技术

Fast bin为单向链表，结构简单，容易伪造。并且为了提高效率，安全检查较少。但是只针对fast bin大小的chunk，对small/large chunk不适用。

利用思路：

  1. 空闲Fast chunk如果发生溢出而被覆盖，则链表指针fd可以被修改。
  2. 可以通过修改链表指针fd，在Fast bin链表中引入伪造的空闲Fast chunk。
  3. 下次分配时分配出伪造的Fast chunk。
  4. 伪造的Fast chunk可以在.bss全局变量处，也可以在栈上。

在栈上伪造Fast chunk：覆盖返回地址。

在bss上伪造Fast chunk，修改全局变量。

在堆上伪造Fast chunk，修改堆上的数据。
